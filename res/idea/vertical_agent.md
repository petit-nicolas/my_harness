## 网文/世界观构建工坊
维护一个小说/故事的完整世界观数据库，帮助写作时保持一致性。

为什么不是 skill 能做的事？

持久化的世界观数据库：角色档案、地点、事件时间线、势力关系、设定规则
character_graph — 可视化人物关系图谱（Streamlit 渲染）
timeline_view — 故事时间线可视化，标注每个事件涉及的角色
consistency_check — 写完一章后自动检查是否和已有设定矛盾
write_chapter — 在已有世界观约束下生成章节草稿
what_if — "如果角色 A 在第 3 章死了，后续剧情会怎么变？"
价值：长篇创作者的核心痛点就是"设定崩了"。写到第 50 章忘了第 3 章的某个细节。这不是 skill 能解决的 — 需要一个真正的数据库来维护世界观状态。

## 算法可视化实验室
// 可以同时考虑物理，数学的可视化教学的素材准备
用对话描述算法，Agent 生成代码并实时渲染算法执行过程的动画。

为什么不是 skill 能做的事？

visualize_sort / visualize_tree / visualize_graph — 预置经典数据结构的可视化渲染器
逐步执行模式 — 可以暂停、单步、回退，查看每一步的状态变化
代码和可视化并排显示 — 当前执行到哪一行，对应数据结构长什么样
compare_algorithms — 同一数据集，两种算法并排跑，对比性能
自动生成复杂度分析报告
价值：学习算法的最好方式是"看到它在动"。这个方向对 CS 学生价值极大。

## SVG 矢量创作工坊
用对话生成和迭代 SVG 图形 — Logo、图标、插画、信息图。

为什么不是 skill 能做的事？ Skill 只能让模型输出 SVG 代码，你得自己复制到浏览器看效果。而垂直 Agent 有：

左侧实时 SVG 渲染面板，改一行代码立刻看到变化
shape_library 工具 — 预置 200+ 基础图形/图案组件，模型通过组合而非从零画
layer_manager — 图层管理，可以对单独图层对话修改
export_variants — 一次导出多尺寸（favicon、App icon、社交媒体头图）
持久化的设计版本历史 — 可以回溯任意迭代
价值：独立开发者/自媒体博主经常需要简单的矢量图形，不想开 Figma 或 Illustrator。一句 "给我画一个极简风格的咖啡杯图标，线条粗细 2px，暖色调" 就能出图并迭代。

## Playwright 网页自动化台
用 Agent 驱动一个真实浏览器，做网页操作自动化、测试、数据采集。

Agent 包装后：

"帮我每天早上 9 点截图这三个竞品的定价页面"
"登录这个后台，导出上个月的销售报表"
"帮我测试这个网页在不同分辨率下的显示效果"
领域工具：

open_page — 打开 URL，返回截图
interact — 点击/输入/滚动等操作
screenshot — 全页截图或元素截图
extract_data — 提取页面数据到结构化格式
record_workflow — 记录操作序列，可重放
Python 库：playwright（微软出品，比 Selenium 更现代）

Streamlit 预览：浏览器截图实时展示 + 操作回放
